# 资源释放

## 必须释放的资源
1. **KcpApplicationPacket** - 每次接收后必须调用`Dispose()`
2. **KcpConnection** - 连接结束时调用`Dispose()`
3. **UdpClient** - 手动管理生命周期

## 为什么只有Dispose()？
FaGe.Kcp的内部资源释放是同步操作，没有需要异步清理的资源（如文件句柄、数据库连接等）。
因此只提供`Dispose()`方法，简化API设计。

```csharp
// 简单直接的释放
connection.Dispose();
```
